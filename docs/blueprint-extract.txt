1. PLATFORM VISION & OVERVIEW 
1.1 What We Are Building 
TechStar UjuziLab (branded externally as UjuziHub) is Africa's modern learning and innovation 
ecosystem. It is NOT a simple LMS. It is a convergence of six systems: 
• LMS — Courses, lessons, quizzes, assignments, certificates 
• Creator Platform — Instructor tools, revenue, analytics 
• Organization Platform — Schools, universities, companies, hubs 
• Innovation Ecosystem — Projects, prototypes, labs, competitions 
• Community Platform — Discussions, mentorship, hackathons 
• Modern SaaS Application — Subscriptions, billing, analytics 
 
1.2 Target Users 
Students / Learners Browse, enroll, learn, complete, get certified 
Instructors / Creators Create courses, upload content, earn revenue 
Organization Admins Manage institution, members, private programs 
Moderators Review content, handle reports, approve courses 
Super Admins Full platform control, analytics, configuration 
Public Visitors Browse catalog, view landing pages, sign up 
 
1.3 Core Business Model 
• Individual paid course purchases 
• Free courses (lead generation) 
• Organization licensing (bulk seats) 
• Bootcamp enrollment fees 
• Certification program fees 
• Subscription plans (Phase 2) 
• Sponsored learning programs 
• Kit / hardware sales (Phase 2) 
• Startup incubation fees (Phase 3)

28. SECURITY & RLS POLICIES 
28.1 Key RLS Policies 
profiles — SELECT auth.uid() = id OR is_public = true 
profiles — UPDATE auth.uid() = id 
courses — SELECT status='published' AND visibility='public' OR (enrolled + org) 
courses — INSERT/UPDATE auth.uid() = instructor_id OR has role instructor/admin 
lessons — SELECT User enrolled in course OR lesson.is_free_preview 
enrollments — SELECT auth.uid() = user_id 
enrollments — INSERT auth.uid() = user_id 
lesson_progress — ALL auth.uid() = user_id 
quiz_attempts — ALL auth.uid() = user_id 
payments — SELECT auth.uid() = user_id OR has role admin 
notifications — ALL auth.uid() = user_id 
admin tables has role super_admin 
moderation tables has role moderator OR super_admin 
 
28.2 Additional Security 
• All Edge Functions validate JWT from Authorization header 
• Selcom webhook: verify HMAC-SHA256 signature before processing 
• Input validation with Zod on all forms and API inputs 
• XSS prevention: Tiptap sanitizes HTML output 
• Rate limiting: Supabase edge function rate limit + Cloudflare rules 
• CORS: restrict to production domain 
• File upload: validate MIME type server-side, not just extension 
• SQL injection: 100% parameterized via Supabase client (never raw SQL in Edge Functions 
without parameterization)
